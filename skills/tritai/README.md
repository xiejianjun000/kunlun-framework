## 💡 两种使用方式

三才提供两种完全独立的使用方式，适用于不同场景：

### 🤖 方式一：代码引擎模式（开发者）

集成到你的 Node.js / Python / Java 项目中，流式守护 LLM 输出，零 Token 消耗。

**适用于**：Agent 系统、API 服务、自动化工作流

### 📋 方式二：纯文本模式（所有人）

不需要任何代码！直接复制 [`WFGY-2.0-CORE.txt`](WFGY-2.0-CORE.txt) 粘贴到任何 LLM 对话窗口，立刻获得防幻觉能力。

**适用于**：网页版 ChatGPT、Claude、Gemini、日常写作、研究工作

> 详细方法论文档：[`docs/WFGY-METHODOLOGY.md`](docs/WFGY-METHODOLOGY.md)

## 🚀 5 分钟快速开始

### 安装

```bash
npm install tritai
```

### 基础用法

```javascript
const { ZeroTokenGuard } = require('tritai');

const guard = new ZeroTokenGuard();

// 检测文本幻觉
const result = guard.detectText('根据 GB-2025-003 第7.2条规定...');

if (result.detected) {
  console.log('🚨 检测到幻觉！');
  console.log('   置信度:', (result.overallConfidence * 100).toFixed(0) + '%');
  console.log('   证据链:', result.evidenceChain);
}
```

### 流式集成（OpenAI 示例）

```javascript
const stream = await openai.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: '环保法规定...' }],
  stream: true,
});

for await (const chunk of choices) {
  const token = chunk.choices[0].delta.content;
  
  // 每个 Token 经过 WFGY 守护
  const warning = guard.processToken(token);
  if (warning) {
    console.log(warning); // 发现幻觉，立刻输出警告
    break; // 可以选择中断后续输出
  }
  
  process.stdout.write(token); // 正常透传
}
```

## 📋 内置检测规则

| 规则 ID | 名称 | 检测原理 | 基础置信度 |
|---------|------|---------|-----------|
| R001 | 假标准编号 | GB/HJ/ISO 标准编号格式校验 + 未来年份 | 95% |
| R003 | 时间穿越 | 未来年份 + 法规/标准关键词 | 88-95% |
| R004 | 自相矛盾 | 正反义词汇同时出现 | 92% |
| R005 | 不存在的历史法规 | 不存在的法律名称黑名单 | 90% |

> **确证原则**：2 条以上规则同时命中时，置信度额外 +15%，最高可达 98%。

## 📖 设计哲学

> 太极生两仪，两仪生四象，四象生八卦。

我们不追求让 AI 变得"全知全能"，那不现实，也不必要。

我们追求的是让 AI **"知之为知之，不知为不知"**：
- ✅ 知道的，就准确地说
- ⚠️ 不确定的，就标明置信度
- ❌ 不知道的，就诚实地说不知道

这就是可信 AI 的基础，也是三才的使命。
