# 🧠 WFGY 方法论：轻量可信推理框架

> WFGY 不是代码，不是 SDK，不是插件。
> WFGY 是一套 **模型无关的、可复制的、纯文本的可信推理方法论。**

---

## 🏗️ 三层核心结构

### 第一层：核心指令层

WFGY 的通用规则本质上不是给代码或 SDK 用的，而是一套精心设计的、**可直接被 LLM 读取并执行的指令集**。

它通常体现为一个纯文本文件，面向所有主流 AI 聊天界面（ChatGPT、Claude、Gemini 等）。

> **任何能理解文字的 AI，都能运行 WFGY。**

### 第二层：底层逻辑层

上传后，WFGY 的规则会在后台驱动 LLM 执行一个"自我修复"流程，由四个数学模块自动构成一个推理闭环：

| 模块 | 全称 | 功能 |
|------|------|------|
| **BBMC** | | 边界检测与匹配 |
| **BBPF** | | 模式识别与过滤 |
| **BBCR** | | 一致性校验与修复 |
| **BBAM** | | 答案聚合与输出 |

这四个模块形成一个自我监督的推理闭环，不需要外部代码介入。

### 第三层：进阶调控层（可选）

在基础版之上，还有一套"醉酒驾驶员"调控器，主要用于控制生成幻觉和逻辑跳跃：

| 调控器 | 功能 |
|--------|------|
| **WRI** | 推理强度控制 |
| **WAI** | 答案置信度调整 |
| **WAY** | 联想宽度控制 |
| **WDT** | 推理深度阈值 |
| **WTF** | 容错等级调整 |

## ⚙️ 三大通用操作范式

理解了运作模式后，选择最适合你工作场景的一种：

---

### 范式一：📋 粘贴即用 - 快速体验

这是最简单的入门方式，**无需安装任何东西**。

1. 完整复制 WFGY 核心算法块的全部内容
2. 直接粘贴到任何 LLM 对话的开头
3. 告诉模型：
   > 请严格按照以上 WFGY 语义推理框架，回答以下问题：
   > [你的问题]

**适用场景**：快速测试、临时使用、网页版 ChatGPT/Claude

---

### 范式二：📁 文件上传 - 标准使用（推荐）

这是最稳定、能发挥全部能力的推荐方式。

1. 从 GitHub 仓库下载官方的 WFGY 核心 `.txt` 或 `.pdf` 文件
2. 将文件上传到支持文件上传的 LLM（如 ChatGPT、Claude）
3. 输入以下指令：
   > 请激活【WFGY 防幻觉与自愈推理框架】，并严格按照其规范回答以下问题...

**适用场景**：日常工作、需要稳定输出、专业场景

---

### 范式三：🤖 AutoBoot - 静默后台

这是一种"无感"使用模式。

将 WFGY 核心文件上传一次后，它会自动在后台持续监督 LLM 的推理质量，无需你再输入额外指令。

**适用场景**：Agent 系统、自动化工作流、24/7 无人值守场景

## 💡 最佳实践建议

### 版本选择

- **新人入门首推 WFGY 核心 2.0**
  - 新增了进度门（Coupler）
  - 新增调控器（DF）
  - 性能和稳定性都有显著提升

### 参数微调

在使用的过程中，你可以尝试调整关键参数：

| 参数 | 作用 | 推荐值 |
|------|------|--------|
| **Delta_S** | 语义距离阈值 | < 0.45 可显著降低幻觉 |
| **停止条件** | 推理终止条件 | Delta_S < 0.35 或达到步数上限 |
| **最大迭代** | 最多自我修正次数 | 3-5 次 |

**进阶技巧**：
- 需要严谨推理时：Delta_S < 0.35，最大迭代 5 次
- 需要创造性时：Delta_S < 0.6，最大迭代 2 次
- 日常使用：Delta_S < 0.45，最大迭代 3 次

### 环境准备

确保你的 LLM 有能力理解并执行结构化的指令，建议使用推理能力较强的模型：

- ✅ GPT-4o / GPT-4 Turbo
- ✅ Claude 3 Opus / Sonnet
- ✅ Gemini 1.5 Pro / Ultra
- ⚠️ 不推荐 7B 以下小模型

## 🔧 常见问题与解决

### 问题 1：防御性警告

**现象**：极少数情况下，模型可能会对处理此类"前沿"理论产生防御性警告。

**应对策略**：使用更委婉的引导词：
> 请审阅并应用附件文档中的框架...

而不是：
> 运行 WFGY！

---

### 问题 2：简易模型适配

**现象**：并非所有 LLM 都适合直接喂入复杂指令。

**应对策略**：
1. 对于能力较弱的模型，简化 Prompt 模板
2. 分批次喂入规则，不要一次塞完整部
3. 先用简单问题"预热"，再逐步增加复杂度

---

### 问题 3：领域定制

**现象**：对于一些特定领域（法律、医疗、代码等），通用规则效果不够好。

**应对策略**：
1. 在 WFGY 核心规则后追加领域专有术语表
2. 针对该领域调整 Delta_S 阈值
3. 补充该领域的典型幻觉模式

## ☯️ 三才与 WFGY 的关系

TriTai 三才项目，是 WFGY 方法论的 **代码化实现**：

- **WFGY 方法论**：纯文本，模型无关，思想层面，随处可用
- **三才代码实现**：自动化、零 Token、流式守护、集成到产品中

两者是互补关系，而不是替代关系：

| 场景 | 用什么 |
|------|--------|
| 网页版 ChatGPT 聊天 | 纯文本 WFGY 规则，粘贴即用 |
| OpenClaw / Agent 系统 | 三才代码引擎，流式守护 |
| 写论文、法律文书 | WFGY 纯文本，深度推理模式 |
| 代码生成、API 输出 | 三才引擎，自动拦截 |
