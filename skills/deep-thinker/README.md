# Deep Thinker - 深度思考能力技能

## 快速开始

Deep Thinker 是一个为 OpenClaw (小龙虾) 设计的深度思考技能，帮助 AI 从简单的"信息提炼"升级到真正的"洞察生成"。

### 安装

```bash
# 将技能复制到 openclaw skills 目录
cp -r deep-thinker ~/.openclaw/skills/

# 或者使用 openclaw CLI
openclaw skill install ./deep-thinker
```

### 使用

**自动触发**（推荐）：
- 当你说"分析一下"、"怎么看"、"为什么"时自动调用
- 当你说"总结思考"、"深度分析"时自动调用

**手动调用**：
```
用户: 用 deep-thinker 深度分析一下我们的用户流失问题
用户: 从系统性思维角度分析这个决策
```

### 控制输出深度

```
# 简洁模式（1个方法 + 2-3个洞察）
用户: 快速思考一下这个问题

# 标准模式（默认，2个方法 + 3-5个洞察）
用户: 分析这个问题

# 深度模式（3-4个方法 + 5-8个洞察 + 详细分析）
用户: 深度分析、全面思考、系统性分析
```

## 核心特性

### 8种思考方法论

| 方法 | 适用场景 | 关键词 |
|------|---------|--------|
| **第一性原理** | 创新、突破性思考 | 本质、基本事实、重新推导 |
| **系统性思维** | 复杂问题、组织问题 | 系统、反馈回路、杠杆点 |
| **批判性思维** | 评估观点、决策支持 | 证据、假设、偏见、逻辑 |
| **结构化思维** | 方案设计、沟通表达 | MECE、框架、拆解 |
| **设计思维** | 用户问题、产品创新 | 同理心、原型、迭代 |
| **逆向思维** | 创新、找问题 | 反转、反向、反面 |
| **类比思维** | 理解概念、解释 | 类比、映射、跨领域 |
| **二阶思维** | 决策评估、长期规划 | 后果的后果、连锁反应 |

### 智能方法选择

技能会根据问题类型自动选择最合适的方法：

- **分析类问题** → 系统性思维 + 批判性思维
- **决策类问题** → 第一性原理 + 二阶思维
- **创新类问题** → 第一性原理 + 设计思维 + 逆向思维
- **问题解决** → 系统性思维 + 逆向思维

## 使用示例

### 示例 1：商业分析

```
用户: 为什么我们的用户留存率一直在下降？深度分析一下

Deep Thinker 输出:
# 🧠 深度思考分析

## 📌 问题理解
- **核心问题**：用户留存率持续下降的根本原因
- **问题类型**：分析类 + 问题解决
- **思考方法**：系统性思维 + 二阶思维

## 🔍 系统性思维分析
[系统要素识别]
[反馈回路分析]
[杠杆点识别]

## 💡 关键洞察
1. 营销-产品错配是核心问题
2. 恶性循环已经形成
3. 获客成本隐性上升

## 🎯 可行动建议
[短期行动]
[中长期策略]
[需要避免的陷阱]
```

### 示例 2：技术决策

```
用户: 我们应该选择 React 还是 Vue？请深度思考

Deep Thinker 会从以下角度分析:
- 第一性原理：问题本质是什么？
- 二阶思维：不同选择的长期影响
- 关键洞察：技术选型的本质是"风险最小化"
- 决策框架：根据团队和项目特征选择
```

### 示例 3：产品创新

```
用户: 如何设计一个真正好用的待办事项应用？

Deep Thinker 会使用:
- 设计思维：用户真实痛点是什么？
- 第一性原理：任务管理的本质是什么？
- 逆向思维：为什么大部分待办应用都失败了？
- 类比思维：可以借鉴什么其他领域？
```

## 输出格式说明

Deep Thinker 的输出始终包含以下部分：

1. **问题理解**：明确核心问题和思考方法
2. **多维度分析**：使用选定的思考方法进行深度分析
3. **关键洞察**：提炼出2-5个非显而易见的发现
4. **可行动建议**：短期行动 + 长期策略 + 避免的陷阱
5. **后续思考方向**：引申问题和需要进一步了解的信息

## 技能原理

### 为什么需要深度思考技能？

大部分 AI 能够快速处理信息，但往往停留在：
- ✗ 简单概括总结
- ✗ 列举显而易见的因素
- ✗ 给出通用建议

Deep Thinker 通过系统化的思考框架，实现：
- ✓ 揭示被忽视的关键因素
- ✓ 发现深层的结构性问题
- ✓ 提出反直觉的观点
- ✓ 预见意料之外的后果

### 洞察 vs 信息

| 信息整理 | 洞察生成 |
|---------|---------|
| "用户流失率高" | "营销-产品错配导致预期落差" |
| "React更流行" | "技术选型的本质是风险最小化" |
| "需要优化性能" | "性能瓶颈在数据库查询，不是代码" |

## 常见问题

### Q: 什么时候应该使用 Deep Thinker？

**应该使用**：
- 需要深入分析复杂问题
- 需要做出重要决策
- 需要创新的解决方案
- 需要理解问题的本质

**不需要使用**：
- 简单的事实查询
- 明确的操作步骤
- 基础的信息整理

### Q: 如何获得更好的分析结果？

1. **提供充分的背景信息**
   - 错误："为什么用户流失？"
   - 正确："我们做B2B SaaS，Q2开始用户流失率从15%上升到25%，主要发生在试用期结束后"

2. **明确你的目标**
   - 错误："分析这个问题"
   - 正确："我们需要决定是否要重构整个系统，请深度分析"

3. **提供相关数据**
   - 如果有数据、报告、链接，一并提供
   - 告诉技能哪些信息是确定的，哪些是推测

### Q: 输出太长怎么办？

使用简洁模式：
```
用户: 快速思考一下，不要展开太多
用户: 简要分析，给出3个关键洞察即可
```
