# Deep Thinker 快速参考

## 一句话说明
从"信息整理"到"洞察生成"的系统化思考框架。

## 核心价值
不只是告诉你"是什么"，而是告诉你"为什么"、"怎么看"、"怎么办"。

---

## 8种思考方法速查

| 方法 | 何时使用 | 关键提问 |
|------|---------|---------|
| **第一性原理** | 创新、突破 | 本质是什么？基本事实是什么？ |
| **系统性思维** | 复杂问题 | 系统要素？反馈回路？杠杆点？ |
| **批判性思维** | 评估观点 | 证据可靠吗？假设成立吗？ |
| **结构化思维** | 方案设计 | 如何MECE拆解？用什么框架？ |
| **设计思维** | 用户问题 | 真实痛点是什么？如何快速验证？ |
| **逆向思维** | 创新、找问题 | 如何让情况变糟？反面是什么？ |
| **类比思维** | 理解概念 | 这像什么？其他领域怎么解决？ |
| **二阶思维** | 决策评估 | 后果的后果是什么？长期影响？ |

## 使用方式

### 自动触发（推荐）
说这些词会自动调用：
- "分析一下"、"怎么看"、"为什么"
- "总结思考"、"深度分析"
- "应该怎么做"、"有什么建议"

### 手动调用
- "用 deep-thinker 分析"
- "从系统性思维角度思考"
- "用第一性原理分析"

### 控制深度
- "快速思考" → 1个方法 + 2-3个洞察
- "分析" → 2个方法 + 3-5个洞察（默认）
- "深度分析" → 3-4个方法 + 5-8个洞察

---

## 输出结构

每次输出包含：
1. **问题理解**：核心问题 + 思考方法
2. **多维度分析**：按方法框架分析
3. **关键洞察**：非显而易见的发现
4. **可行动建议**：短期 + 长期 + 避免的陷阱
5. **后续思考方向**：引申问题

---

## 洞察 vs 信息

| ❌ 不是洞察 | ✅ 是洞察 |
|------------|----------|
| 重复已知信息 | 指出被忽视的因素 |
| 简单概括 | 揭示深层结构 |
| 明显因果 | 反直觉观点 |
| 常规建议 | 非显而易见的方案 |

## 典型场景

### 商业分析
```
用户: 为什么用户留存率下降？
Deep Thinker: 系统性思维分析 → 恶性循环 → 杠杆点
洞察: 营销-产品错配、预期管理失败
```

### 技术决策
```
用户: React 还是 Vue？
Deep Thinker: 第一性原理 + 二阶思维
洞察: 技术选型本质是"风险最小化"，不是"选最好的"
```

### 产品创新
```
用户: 如何设计好用的待办应用？
Deep Thinker: 设计思维 + 逆向思维
洞察: 大部分待办应用失败的原因是"管理欲望 > 执行动力"
```

## 获取更好结果的技巧

1. **提供背景信息**
   - ✅ "我们做B2B SaaS，用户流失率从15%升到25%"
   - ❌ "为什么用户流失？"

2. **明确目标**
   - ✅ "需要决定是否重构系统"
   - ❌ "分析这个问题"

3. **提供数据**
   - 如果有数据、报告、链接，一并提供

## 常见问题

**Q: 什么时候应该用 Deep Thinker？**
- 需要深入分析复杂问题
- 需要做重要决策
- 需要创新解决方案
- 不需要：简单查询、基础信息整理

**Q: 输出太长怎么办？**
说："快速思考"、"简要分析"

**Q: 可以指定方法吗？**
可以："用第一性原理分析"、"从系统性思维角度"

**Q: 可以指定视角吗？**
可以："从用户视角"、"从商业和技术双视角"

## 与普通AI回答的区别

| 普通回答 | Deep Thinker |
|---------|--------------|
| 告诉你"是什么" | 告诉你"为什么"、"怎么办" |
| 信息整理 | 洞察生成 |
| 表面总结 | 深层分析 |
| 单一视角 | 多维度交叉验证 |
| 常规建议 | 可行动策略 + 避免陷阱 |
