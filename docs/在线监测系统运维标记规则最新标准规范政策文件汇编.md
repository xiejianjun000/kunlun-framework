## 二、HJ 212-2025（最核心标准）

### 2.1 核心定位

> **标准号：** HJ 212—2025
> **代替：** HJ 212—2017
> **发布时间：** 2025年6月6日
> **实施时间：** 2026年1月1日
> **性质：** 国家生态环境标准（强制性/推荐性待确认）

### 2.2 核心变化：从"在线监控"升级为"自动监测+智能监控"一体化

```
HJ 212-2017（2017版）：
  仅关注：污染源自动监测数据
  手段：数据传输协议规范化
  数据：仅有污染物浓度

HJ 212-2025（2026新版）：
  关注：污染物监测 + 用电监控 + 工况识别 + 视频监控 + 数据传输
  手段：一体化数据采集传输
  数据：多维监管数据融合
```

**五大升级维度：**

| 维度 | HJ 212-2017 | HJ 212-2025 |
|------|-------------|-------------|
| 数据类型 | 污染源浓度数据 | 污染源+用电+工况+视频+传输 |
| 传输频率 | 分钟级 | 实时/准实时（<=10s废气） |
| 数据标记 | 基础状态标记 | 智能标记（自动判定） |
| 数据有效性 | 人工审核 | 附录G自动计算 |
| 系统架构 | 单点数据采集 | 一体化智能监控体系 |

### 2.3 关键条款

**数据标记条款：**
> "数据标记由现场端根据仪器仪表提供的数据标记、运行参数自动判定并与监测数据同步上传，**不应人工干预**。数据传输有效性计算规则参见附录G。"

**表10 自动监测设备标记内容表：**
```
标记内容及代码：
  数据有效（N）    ← 默认状态
  停运（Td）      ← 设备停运时标记
  故障（T）        ← 设备故障时标记
  维护（M）        ← 日常维护时标记
  数据补全（C）    ← 数据补全时标记
  工况异常（D）    ← 非正常工况时标记

优先级（从高到低）：D > C > M > T > Td > N
  当同一时间有多个标记时，按优先级自动认定
```

**废气数据有效性判定规则：**
```
标记为"停运"的，应同时满足：
  废气氧含量 > 20%（或氧含量不低于当地基准氧含量）
```

**废水数据有效性判定规则：**
```
废水污染物（化学需氧量、总有机碳等）数据有效性判定：
  a） 排除异常值后计算
  b） 须符合排放标准和测量方法要求
```

## 三、生态环境监测条例（国务院令第820号）

### 3.1 核心定位

> **发布时间：** 2025年11月6日（李强签署）
> **施行时间：** 2026年1月1日
> **历史意义：** 我国首部生态环境监测领域行政法规，填补了监测法律空白

### 3.2 关键条款

**数据质量保障（核心）：**
```
第四十八条：生态环境监测数据安全保护，依照
《中华人民共和国数据安全法》、《网络数据安全管理条例》
的规定执行。
```

**自行监测义务：**
- 企事业单位必须按照排污许可证要求开展自行监测
- 监测数据必须真实、准确、完整
- 监测机构对数据质量负责

**法律责任：**
```
禁止行为：
  - 擅自删除、篡改、伪造监测数据
  - 以逃避监管方式排放污染物
  - 未按规定安装、使用自动监测设备

处罚：
  - 行政处罚 + 信用惩戒
  - 严重者追究刑事责任（污染环境罪）
```

**对社会监测机构的要求：**
- 监测机构应当取得资质认定
- 禁止出具虚假监测报告
- 纳入信用管理体系

## 四、污染物排放自动监测设备标记规则

> **发布：** 生态环境部公告2022年第21号
> **发布时间：** 2022年7月19日
> **适用范围：** 所有行业

### 4.1 三类标记详解

**① "自动监测设备维护"标记（M）**

触发条件：
- 自动监测设备调试
- 故障
- 日常维护
- 校准
- 导致数据缺失或异常的其他情况

标记要求：
```
数据缺失 → 需在缺失时段内进行设备维护标记
数据异常 → 需在异常发生时段内进行标记
标记应与监测数据同步上传，不得事后补录
```

**② "数据补全"标记（C）**

触发条件：
- 设备故障期间的数据需要补全时
- 数据缺失需要补录时
- 特殊情况下的数据补充

要求：
```
数据补全必须基于实际测量或经验证的方法
严禁虚构数据
补全数据须标注来源
```

**③ "数据有效"标记（N）**

- **默认状态**：自动监测数据默认自动标记为"数据有效"
- 不需要人工干预
- 与监测数据同步上传

### 4.2 工况标记规则

触发条件：
- 生产设施启动/停车
- 工况异常（非正常工况）
- 生产工艺变化

工况标记规则：
```
工况标记：由生产设施的实际运行状态决定
不是数据有效性问题
而是帮助判断"为什么数据偏高/偏低"的背景信息
```

### 4.3 数据有效性认定优先级

```
当同一时段有多个标记时，按以下优先级认定：

优先级从高到低：
  D（工况异常）> C（数据补全）> M（维护）> T（故障）> Td（停运）> N（有效）

示例：
  同一天既标记了"故障"又标记了"维护"
  → 系统自动按优先级高者（D）认定
```

## 五、武汉市污染源自动监控管理办法（政府令第330号）

> **发布时间：** 2025年12月24日
> **施行时间：** 2026年4月1日
> **适用范围：** 武汉市行政区域内重点排污单位

### 5.1 核心框架

```
第一章：总则（1-3条）→ 目的、依据、适用范围
第二章：安装与联网（4-9条）→ 设备选型、安装、调试、验收、联网
第三章：运行与维护（10-14条）→ 运维管理、台账要求
第四章：数据应用（15-17条）→ 非现场监管、数据应用
第五章：监督管理（18-25条）→ 禁止行为、法律责任
第六章：附则（26-27条）→ 定义
```

### 5.2 安装与联网时限要求

| 企业类型 | 完成时限 |
|---------|---------|
| 重点排污单位（重点排污许可管理） | 取得排污许可证后**3个月内** |
| 重点排污单位（重点排污单位名录）| 名单公布后**6个月内** |

**联网要求：**
```
安装调试完成 → 验收 → 5个工作日内向生态环境局登记
↓
登记表上传至监控平台
↓
数据传输有效率须达到相关要求
```

### 5.3 运维管理要求（关键条款）

**第十条（运维方式）：**
```
排污单位可自行运维或委托第三方运维单位进行运维
运维单位应配备足够人员和备品备件
建立相关管理制度，确保运维质量
```

**第十一条（运维内容）：**
```
定期维护
校准
比对监测
确保自动监测设备正常运行
有效传输率达标
```

**第十二条（故障处理）：**
```
故障发生后 → 12小时内向生态环境局报告
↓
5个工作日内修复
↓
恢复正常运行
```

**第十三条（台账管理）：**
```
排污单位应建立自动监测设备电子化运维台账
技术档案
原始监测记录
设备运行参数
运维记录
↓
实时上传至监控平台
```

**第十四条（保存期限）：**
```
台账记录保存期限：≥5年
```

### 5.4 数据应用（非现场监管体系）

**第十五条：**
> "生态环境主管部门应当建立健全以污染源自动监控为主的非现场监管执法体系"

**第十六条（超标判定）：**
```
日均值超标判定：
  日均值 = 当日各有效小时值（或日均监测值）的算术平均值
  日均值 > 排放标准限值 → 认定为超标排放

pH值判定：
  按相关标准执行
```

**第十七条（现场检查）：**
```
拒绝接受检查的
篡改、伪造监测数据的
↓
依法追究责任
```

### 5.5 禁止行为与法律责任（关键条款）

**第十八条（禁止行为）：**
```
禁止：破坏、干扰自动监控系统正常运行
禁止：篡改、伪造监测数据
↓
可举报
```

**第十九条（未按规定安装联网）：**
```
认定为未按规定安装联网的情形：
  1. 未安装自动监测设备
  2. 未联网
  3. 未通过验收
  4. 未按时报送登记材料
```

**第二十条（未确保正常运行）：**
```
认定为未确保自动监测设备正常运行的情形：
  1. 未建立运维台账
  2. 未实时上传运维记录
  3. 数据有效传输率未达标
  4. 故障未及时报告
  5. 设备未在规定时限内修复
```

**第二十一条（数据标记造假）：**
```
认定为"通过篡改数据标记逃避监管"的违法行为：
  1. 设备故障产生的数据，擅自标记为有效数据
  2. 数据标记为非正常工况，标记时段内的数据与标记内容明显不匹配
  3. 其他虚假标记行为
↓
按篡改监测数据论处
视为"通过逃避监管方式排放污染物"
```

**第二十三条（信用惩戒）：**
```
行政处罚决定 → 记入信用信息系统 → 向社会公开
```

def compute_data_validity(marking_records: list, timestamps: list) -> float:
    """
    数据有效传输率计算
    HJ 212-2025 附录G
    
    公式：
    数据有效传输率 = 有效数据个数 / 应传输数据个数 × 100%
    
    有效数据：
      标记为"N"（数据有效）的数据
      标记为"C"（数据补全）但经核实的补全数据
    
    无效数据：
      标记为"T"（故障）
      标记为"Td"（停运）
      标记为"M"（维护）
      超出测量范围的数据
    """
    total = len(timestamps)
    valid = 0
    
    for record in marking_records:
        if record.mark == 'N':
            valid += 1
        elif record.mark == 'C' and record.verified:
            valid += 1
        elif record.mark in ['T', 'Td', 'M']:
            continue  # 无效数据
        elif record.value > record.range_max or record.value < record.range_min:
            continue  # 超范围数据
        else:
            valid += 1
    
    return valid / total if total > 0 else 0
