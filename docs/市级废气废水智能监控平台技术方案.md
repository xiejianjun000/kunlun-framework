## 一、项目背景与目标

### 1.1 需求分析

市级生态环境局需要在辖区内的重点排污企业安装智能监控 Agent，对以下四类违规行为进行**实时识别与预警**：

| 监控目标 | 具体行为 | 违规本质 |
|---------|---------|---------|
| **超标排放** | 废气/废水排放浓度超出国家标准 | 数据正常但超标 |
| **作弊软件** | 工控机上安装软件篡改监测数据 | 数据被人为伪造 |
| **参数篡改** | 运维人员修改工控机参数降低监测灵敏度 | 数据被人为压低 |
| **运维造假** | 运维人员不按时更换标气/通标气 | 运维记录虚假 |

### 1.2 技术挑战

```
挑战1：超标排放识别
  → 难点：国家标准限值随行业/工段/时段变化
  → 方法：法规知识图谱 + 时序异常检测

挑战2：作弊软件检测
  → 难点：工控机上软件行为不透明，隐蔽性强
  → 方法：ICS流量异常检测 + 进程行为分析

挑战3：参数篡改检测
  → 难点：运维人员有合法操作权限，难以区分
  → 方法：操作审计 + 参数基线偏离检测

挑战4：运维造假检测
  → 难点：标气钢瓶更换无自动记录
  → 方法：标气使用量时序分析 + 运维记录交叉验证
```

## 三、四大核心功能模块设计

### 3.1 模块一：超标排放识别（ExceedanceDetector）

**核心原理：** 时序异常检测 + 法规知识图谱

```
监测数据流（每分钟）
    ↓
时序异常检测（Isolation Forest + Autoencoder）
    → 检测异常浓度波动
    ↓
法规知识图谱查询（EcoLawGraph）
    → 获取当前企业适用的：
      · 行业排放标准（GB/行业标准）
      · 工段工艺标准
      · 时段限值（早/晚高峰）
      · 特殊天气应急管控要求
    ↓
超标判定
    → 三级预警：提醒(>80%) / 警告(>100%) / 严重(>150%)
    ↓
自动生成执法建议
    → 适用法条（大气法第48条等）
    → 裁量基准
    → 类似案例
```

**参考技术：**
- AQI异常检测系统（devinxhacker）：Isolation Forest + LOF + Autoencoder 三算法融合
- ICS异常检测库（pwwl/ics-anomaly-detection）：重建误差阈值法

**关键代码设计：**

```python
class ExceedanceDetector:
    """
    基于时序异常检测 + 法规知识图谱的超标识别
    参考：AQI异常检测系统 + ICS异常检测
    """
    
    def __init__(self, law_graph: EcoLawGraph, influx_client):
        # 时序异常检测模型（三种算法融合）
        self.iforest = IsolationForest(n_estimators=200, contamination=0.01)
        self.autoencoder = build_autoencoder(input_dim=n_features)
        self.lof = LocalOutlierFactor(n_neighbors=20, contamination=0.01)
        
        # 法规知识图谱（查询排放标准）
        self.law_graph = law_graph
        
        # 时序数据库
        self.influx = influx_client
        
    def detect(self, enterprise_id: str, readings: dict) -> ExceedanceAlert:
        # Step 1: 时序异常检测
        anomaly_scores = self._compute_anomaly_scores(readings)
        
        # Step 2: 法规知识图谱查询
        standards = self.law_graph.query_standards(
            enterprise_id=enterprise_id,
            pollutant=readings['pollutant'],
            timestamp=readings['timestamp']
        )
        
        # Step 3: 超标计算
        exceedance_ratio = readings['value'] / standards['limit']
        
        # Step 4: 风险评分（参考 ICS 异常检测的重建误差评分）
        risk_score = anomaly_scores['avg'] * exceedance_ratio
        
        # Step 5: 关联法条和裁量基准
        applicable_articles = self.law_graph.find_applicable_articles(
            pollutant=readings['pollutant'],
            exceedance_type=standards['type']
        )
        
        # Step 6: 生成执法建议
        recommendation = self._generate_enforcement_recommendation(
            enterprise_id=enterprise_id,
            exceedance_ratio=exceedance_ratio,
            articles=applicable_articles,
            similar_cases=self._find_similar_cases(enterprise_id, readings)
        )
        
        return ExceedanceAlert(
            enterprise_id=enterprise_id,
            exceedance_ratio=exceedance_ratio,
            risk_score=risk_score,
            applicable_articles=applicable_articles,
            enforcement_recommendation=recommendation,
            evidence=self._collect_evidence(readings, standards)
        )
```

### 3.2 模块二：作弊软件检测（CheatSoftwareDetector）

**核心原理：** ICS 流量异常检测 + 进程行为分析

```
工控机流量数据（每秒抓包）
    ↓
ICS 流量分析（参考 HAI 数据集方法）
    → 检测与正常模式的偏差
    → 识别未授权的 Modbus 写指令
    ↓
进程行为监控
    → 新进程启动检测（文件监控）
    → 关键进程状态监控
    → DLL 注入检测
    ↓
软件行为签名匹配
    → 已知作弊软件行为库（向量检索）
    → 异常行为模式识别
    ↓
告警：检测到疑似作弊软件活动
```

**参考技术：**
- HAI 数据集（icsdataset/hai）：4套 ICS 测试床 + 已知攻击类型
- Anthropic SCADA 检测 Skills（mukul975，4513 Stars）：MITRE ATT&CK for ICS

**HAI 数据集攻击类型（可直接用于训练作弊检测模型）：**

| 攻击类型 | 描述 | 在作弊检测中的对应 |
|---------|------|-----------------|
| 注入攻击 | 在传感器数据中注入虚假值 | 监测数据被篡改 |
| 恶意软件 | 在工控机上安装恶意程序 | 作弊软件 |
| 拒绝服务 | 阻断正常通信 | 监测设备掉线 |
| 隐蔽操控 | 缓慢改变参数而不触发告警 | 参数压低（慢慢篡改）|

**关键代码设计：**

```python
class CheatSoftwareDetector:
    """
    基于 ICS 流量异常检测 + 进程行为的作弊软件检测
    参考：HAI 数据集 + Anthropic SCADA Skills（MITRE ATT&CK for ICS）
    """
    
    def __init__(self, pcap_client, process_watcher, vector_db):
        # ICS 流量分析（参考 HAI 数据集）
        # 4种 ICS 协议：Emerson Ovation / GE Mark VIe / Siemens S7-300 / S7-1500
        self.ics_analyzer = ICSFlowAnalyzer(protocols=[
            'modbus_tcp', 'opc_ua', 'dnp3', 'ethernet_ip'
        ])
        
        # 进程行为监控
        self.process_watcher = ProcessWatcher(
            watch_paths=[
                '/opt/SCADA/',      # SCADA 软件目录
                '/usr/local/bin/',  # 可执行文件
                'C:\\SCADA\\',      # Windows 工控机
            ],
            whitelist=self._load_process_whitelist()
        )
        
        # 作弊软件行为向量库（参考向量检索）
        self.vector_db = vector_db  # Qdrant
        
        # MITRE ATT&CK for ICS 映射（来自 Anthropic SCADA Skills）
        self.atlas_mapping = ATTACKAtlasMapper()
        
    def detect(self, enterprise_id: str, 
               pcap_data: bytes, 
               process_snapshot: dict) -> CheatAlert:
        
        # Step 1: ICS 流量异常检测（参考 HAI 数据集方法）
        ics_anomalies = self.ics_analyzer.detect(pcap_data)
        
        # Step 2: 进程行为异常检测
        process_anomalies = self.process_watcher.detect(process_snapshot)
        
        # Step 3: 已知作弊模式向量检索
        # 提取当前行为特征向量
        behavior_vector = self._extract_behavior_vector(
            ics_anomalies, process_anomalies
        )
        
        # Qdrant 向量相似检索
        similar_cheats = self.vector_db.search(
            collection='cheat_patterns',
            query_vector=behavior_vector,
            score_threshold=0.85
        )
        
        # Step 4: MITRE ATT&CK for ICS 战术映射
        tactical_mapping = self.atlas_mapping.map(
            anomalies=[*ics_anomalies, *process_anomalies]
        )
        
        # Step 5: 综合风险评分
        risk_score = self._compute_cheat_risk_score(
            ics_anomalies=ics_anomalies,
            process_anomalies=process_anomalies,
            similar_cheats=similar_cheats,
            tactical_mapping=tactical_mapping
        )
        
        if risk_score > 0.7:
            return CheatAlert(
                enterprise_id=enterprise_id,
                alert_type='CHEAT_SOFTWARE_DETECTED',
                risk_score=risk_score,
                attack_techniques=tactical_mapping,
                evidence={
                    'ics_anomalies': ics_anomalies,
                    'process_anomalies': process_anomalies,
                    'similar_patterns': [c.metadata for c in similar_cheats],
                    'pcap_evidence': self._save_pcap_evidence(pcap_data)
                },
                taiji_skill='执法-作弊软件检测'
            )
```

### 3.3 模块三：参数篡改检测（ParamTamperDetector）

**核心原理：** 参数基线建模 + 操作审计追踪

```
运维操作日志流
    ↓
基线参数识别（历史正常操作数据训练）
    → 每类工控机参数正常范围
    → 每个行业参数典型值
    ↓
参数变更事件捕获
    → 捕获所有写操作（Modbus/OPC UA）
    → 关联操作人身份（工号+指纹/刷脸记录）
    ↓
三重验证
    ① 授权验证：操作人是否有权限修改该参数？
    ② 时序验证：操作时间是否在运维窗口内？
    ③ 趋势验证：参数变更方向是否符合正常运维逻辑？
    ↓
可疑操作标记 + 告警
```

**关键代码设计：**

```python
class ParamTamperDetector:
    """
    参数篡改检测：基于基线偏离 + 操作审计 + 三重验证
    """
    
    def __init__(self, baseline_model, audit_logger, rbac_service):
        # 参数基线模型（从历史正常数据训练）
        self.baseline_model = ParamBaselineModel()
        self.baseline_model.load('models/param_baseline_v1.pkl')
        
        # 操作审计日志
        self.audit_logger = audit_logger
        
        # RBAC 服务（运维权限管理）
        self.rbac = rbac_service
        
        # 参考 ICS 异常检测的时序建模方法
        self.forecasting_model = TFTModel(...)  # Temporal Fusion Transformer
        
    def detect(self, enterprise_id: str, 
               param_change: ParamChangeEvent) -> TamperAlert:
        
        # Step 1: 查基线——该参数正常范围是多少？
        baseline = self.baseline_model.get_baseline(
            enterprise_id=enterprise_id,
            param_name=param_change.param_name,
            process_type=param_change.process_type
        )
        
        # Step 2: 检测偏离幅度
        deviation = abs(param_change.new_value - baseline['mean']) / baseline['std']
        
        # Step 3: 时序预测验证——参数变更是否在预期范围内？
        forecasted_range = self.forecasting_model.predict(
            enterprise_id=enterprise_id,
            param_name=param_change.param_name,
            horizon='7d'
        )
        is_in_forecast = forecasted_range.contains(param_change.new_value)
        
        # Step 4: 三重验证
        auth_ok = self._verify_authorization(param_change.operator, param_change.param_name)
        timing_ok = self._verify_timing_window(param_change.timestamp)
        trend_ok = self._verify_trend_direction(param_change)
        
        # Step 5: 综合判断
        if not auth_ok:
            severity = 'CRITICAL'  # 未授权操作
            risk_score = 1.0
        elif not timing_ok or not trend_ok:
            severity = 'HIGH'  # 可疑操作
            risk_score = 0.8
        elif deviation > 3.0 and not is_in_forecast:
            severity = 'HIGH'  # 严重偏离
            risk_score = 0.75
        else:
            return None  # 正常操作，不告警
        
        # Step 6: 生成取证包
        evidence = self._build_tamper_evidence(
            param_change=param_change,
            baseline=baseline,
            deviation=deviation,
            verification_results={
                'auth': auth_ok,
                'timing': timing_ok,
                'trend': trend_ok
            }
        )
        
        return TamperAlert(
            enterprise_id=enterprise_id,
            alert_type='PARAM_TAMPER',
            severity=severity,
            risk_score=risk_score,
            param_change=param_change,
            evidence=evidence,
            taiji_skill='执法-参数篡改检测'
        )
```

### 3.4 模块四：运维合规检测（MaintenanceVerifier）

**核心原理：** 标气使用量时序分析 + 运维记录交叉验证

```
标气管理数据流
    ├─ 标气钢瓶更换记录（电子标签扫描）
    ├─ 标气使用量（流量计实时数据）
    ├─ 运维打卡记录（人脸识别/指纹）
    └─ 通标气校准记录（DCS 系统日志）
         ↓
时序分析（Autoencoder + Isolation Forest）
    → 识别异常使用量模式
    ↓
交叉验证
    ├─ 更换记录时间 vs 通标气校准时间（时间差 < 30min?）
    ├─ 使用量下降 vs 更换记录（钢瓶真的换了？）
    └─ 运维打卡 vs 通标气操作人（人脸识别确认）
    ↓
运维合规判定
    ├─ 按时且合规 → 通过
    ├─ 按时但不合规 → 警告（形式合规实质违规）
    └─ 未按时 → 严重违规
```

**关键代码设计：**

```python
class MaintenanceVerifier:
    """
    运维合规检测：时序分析 + 交叉验证
    参考：AQI 异常检测的多算法融合方法
    """
    
    def __init__(self, gas_usage_db, opc_client, rbac_service):
        # 标气使用量数据库
        self.gas_usage_db = gas_usage_db
        
        # OPC UA 客户端（读取 DCS 系统校准记录）
        self.opc = opc_client
        
        # 运维权限服务
        self.rbac = rbac_service
        
        # 时序异常检测模型（AQI 系统同款）
        self.anomaly_models = {
            'iforest': IsolationForest(n_estimators=200),
            'autoencoder': build_autoencoder(input_dim=5),  # 5个关键参数
            'lof': LocalOutlierFactor(n_neighbors=15)
        }
        
        # 行业标准通标气周期（来自法规知识图谱）
        self.calibration_cycles = self._load_calibration_standards()
        
    def verify(self, enterprise_id: str, 
               period: tuple) -> MaintenanceReport:
        
        # Step 1: 收集数据
        gas_changes = self._get_gas_change_records(enterprise_id, period)
        gas_usage = self._get_gas_usage_data(enterprise_id, period)
        calibration_records = self._get_calibration_records(enterprise_id, period)
        op_records = self._get_operator_records(enterprise_id, period)
        
        # Step 2: 时序异常检测（参考 AQI 系统）
        anomaly_scores = {}
        for name, model in self.anomaly_models.items():
            scores = model.fit_predict(gas_usage.features)
            anomaly_scores[name] = scores
        
        # 综合评分（三种算法投票）
        final_anomaly = self._ensemble_vote(anomaly_scores)
        
        # Step 3: 交叉验证（核心逻辑）
        verification_results = []
        for change_record in gas_changes:
            cross_check = self._cross_verify(change_record, calibration_records, op_records)
            verification_results.append(cross_check)
        
        # Step 4: 合规判定
        compliance_results = []
        for change, cross in zip(gas_changes, verification_results):
            if not cross['change_record_exists']:
                result = ComplianceResult(
                    status='VIOLATION',
                    type='NO_CHANGE_RECORD',
                    severity='CRITICAL',
                    evidence=cross['evidence']
                )
            elif not cross['time_difference_ok']:
                result = ComplianceResult(
                    status='VIOLATION',
                    type='LATE_CALIBRATION',
                    severity='HIGH',
                    evidence=cross['evidence']
                )
            elif not cross['operator_verified']:
                result = ComplianceResult(
                    status='VIOLATION',
                    type='OPERATOR_MISMATCH',
                    severity='HIGH',
                    evidence=cross['evidence']
                )
            elif not cross['usage_pattern_normal']:
                result = ComplianceResult(
                    status='WARNING',
                    type='ABNORMAL_USAGE',
                    severity='MEDIUM',
                    evidence=cross['evidence']
                )
            else:
                result = ComplianceResult(
                    status='COMPLIANT',
                    type='FULL_COMPLIANCE',
                    severity=None
                )
            compliance_results.append(result)
        
        return MaintenanceReport(
            enterprise_id=enterprise_id,
            period=period,
            overall_status=self._compute_overall_status(compliance_results),
            results=compliance_results,
            anomaly_scores=final_anomaly,
            taiji_skill='执法-运维合规检测'
        )
```
