## 🎯 语言体系设计哲学

### 为什么需要太极语言？

1. **灵魂统一**：整个项目使用同一种语言体系，形成独特的文化认同
2. **减少歧义**：同一个概念永远用同一个词，减少沟通和理解成本
3. **记忆传承**：用大家熟悉的传统文化概念，降低学习门槛
4. **情感连接**：有温度的语言，而不是冷冰冰的技术术语
5. **品牌识别**：独特的语言体系是开源项目最好的品牌标识

### 设计三原则

| 原则 | 说明 |
|------|------|
| **信达雅** | 准确对应技术概念，同时有文化底蕴，朗朗上口 |
| **可搜索** | 即使是太极术语，也能被搜索引擎理解和检索 |
| **可并行** | 太极语言和技术语言可以并行使用，互不冲突 |

## 🎯 使用规范

### 1. 场景适配

不是所有地方都要使用太极语言，请根据场景选择：

| 场景 | 建议语言比例 | 示例 |
|------|------------|------|
| **面向用户的文档** | 太极语言 70% + 技术语言 30% | "行者在施法前会先检查识海容量" |
| **内部架构文档** | 太极语言 50% + 技术语言 50% | "Actor（行者）通过 Mailbox（传音入密）接收消息" |
| **代码核心逻辑注释** | 太极语言 30% + 技术语言 70% | "执行技能（施法），如果失败则重试" |
| **变量命名** | 优先技术命名，可选太极别名 | 主要用 `agent`，可以用 `practitioner` 作为别名 |
| **错误信息** | 太极语言 20% + 技术语言 80% | "技能执行失败（施法失败）：超时" |
| **团队日常沟通** | 太极语言 80% + 技术语言 20% | "这个 PR 过了试炼了吗？什么时候降世？" |

### 2. 并列标注原则

首次出现太极术语时，建议并列标注对应的技术术语，方便新用户理解：

**✅ 正确示范：**
> 每个**行者 (Actor)** 都有自己的**传音入密 (Mailbox)**，通过**消息 (Message)** 进行通信。

**❌ 错误示范：**
> 每个行者都有自己的传音入密，通过消息进行通信。

### 3. 搜索友好原则

在文档的关键位置，一定要保留标准技术术语，确保搜索引擎可以正确索引：

**✅ 正确示范：**
> ## 安全沙箱（结界）
> OpenTaiji 使用沙箱（Sandbox）技术隔离用户代码执行...

### 4. 一致性原则

- 同一个概念，在同一个文档中，永远用同一个词
- 不要一会儿用"行者"，一会儿用"修士"，一会儿用"代理"
- 如果必须切换，要明确说明是同一个概念的不同表达

## 📝 新增术语流程

当需要新增太极术语时，请遵循以下流程：

1. **提出**：在团队中提出需要新术语的技术概念
2. **征集**：征集大家的术语建议，至少 3 个候选
3. **投票**：团队投票选出最贴切的术语
4. **审核**：检查是否符合"信达雅、可搜索、可并行"三原则
5. **登记**：加入本术语表，发布全员通知
6. **推广**：在所有文档和沟通中开始使用

## 🎨 语言风格指南

### 要做到

✅ **有温度**：用有感情的词，而不是冷冰冰的技术术语  
✅ **有画面**：说出来就能让人联想到画面，比如"照妖镜"、"传音入密"  
✅ **有典故**：最好能从传统文化中找到出处，有文化底蕴  
✅ **好记忆**：朗朗上口，容易记住，不要太生僻  

### 要避免

❌ **过度滥用**：不是所有东西都要改，常见术语保持原样大家更习惯  
❌ **生僻难懂**：不要用大家都不认识的字，降低学习门槛  
❌ **歧义太大**：和原概念偏差太大，让人不知所云  
❌ **不够严肃**：太搞笑或太随意的词，不适合正式技术文档
