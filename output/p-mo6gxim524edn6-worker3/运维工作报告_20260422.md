## 🔧 今日问题处理

### ❌ 问题 1：性能测试超时（严重）

**问题描述**:
- 性能测试在 CI 环境中超时（180s）
- 涉及文件: `tests/performance/*.test.ts`（3个文件）
- 测试用例: 包含并发压力测试、内存泄漏检测、大吞吐量验证

**问题分析**:
```
测试文件清单：
├── llm-adapter-stress.test.ts       - LLM 适配器压力测试（50并发×1000次）
├── determinism-stress.test.ts        - 确定性系统压力测试（多场景并发）
├── memory-leak.test.ts               - 内存泄漏检测测试
└── hallucination-detector-deep.test.ts - 深度功能测试
```

**处理建议**:
| 方案 | 说明 | 优先级 |
|------|------|--------|
| **延长超时时间** | 将性能测试 timeout 从 5000ms 调整为 30000ms | 🔴 高 |
| **分离性能测试** | 创建单独的 `test:perf` 命令，不纳入默认 CI 流程 | 🟡 中 |
| **降低测试规模** | CI 环境使用较小的并发数，本地开发才运行完整规模 | 🟡 中 |
| **定时运行** | 性能测试改为每日定时运行（cron），不每次 PR 都执行 | 🟢 低 |
