### 1. BaichuanAdapter (百川智能)

**API 端点**: `https://api.baichuan-ai.com/v1/chat/completions`

| baichuan3-turbo | 0.003 | 0.003 |

**鉴权方式**: Authorization Bearer Token

## 📖 使用示例

### 基本使用

```typescript
import { LLMAdapterFactory, LLMType } from './src/adapters/llm/LLMAdapterFactory';

// 创建百川适配器
const adapter = LLMAdapterFactory.createAdapter(LLMType.BAICHUAN, {
  apiKey: 'your-api-key',
  model: 'baichuan4',
});

// 检查可用性
if (await adapter.checkAvailability()) {
  // 调用 API
  const result = await adapter.createChatCompletion({
    messages: [{ role: 'user', content: '你好' }],
    stream: false,
  });

  console.log('回复:', result.choices[0].message.content);
  console.log('Token 用量:', result.usage);
  console.log('费用:', adapter.calculateCost(result.usage));
}
```

### 流式响应

```typescript
const stream = await adapter.createChatCompletion({
  messages: [{ role: 'user', content: '写一首诗' }],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0].delta.content || '');
}
```

## 🔧 技术亮点

### 1. 统一接口设计

所有适配器都遵循 `BaseLLMAdapter` 抽象类，提供一致的调用体验：

```typescript
abstract class BaseLLMAdapter {
  abstract readonly providerName: string;
  abstract get isReady(): boolean;
  abstract init(config: AdapterConfig): Promise<void>;
  abstract createChatCompletion(request: ChatCompletionRequest): Promise<ChatCompletionResponse>;
  abstract createChatCompletionStream(request: ChatCompletionRequest): AsyncGenerator<ChatCompletionStreamResponse>;
  abstract calculateCost(usage: TokenUsage): number;
  abstract checkAvailability(): Promise<boolean>;
}
```

### 2. 内置重试机制

所有适配器继承基类的 `fetchWithRetry` 方法，自动处理：
- 网络错误重试（最多 3 次）
- 指数退避算法
- 429 速率限制自动等待
- 详细的错误日志

### 3. 类型安全

完整的 TypeScript 类型定义，无 `any` 类型：

```typescript
export interface ChatCompletionRequest {
  model: string;
  messages: ChatMessage[];
  temperature?: number;
  top_p?: number;
  stream?: boolean;
  max_tokens?: number;
}

export interface ChatCompletionResponse {
  id: string;
  object: string;
  created: number;
  model: string;
  choices: Choice[];
  usage: TokenUsage;
}
```

### 4. 灵活的工厂模式

`LLMAdapterFactory` 提供统一的创建入口，支持动态扩展：

