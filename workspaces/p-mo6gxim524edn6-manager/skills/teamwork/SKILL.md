# teamwork Skill

> 提升团队协作的交互体验，让项目经理 能够清晰地向用户汇报任务进度、状态和最终成果。
> 解决用户只能与团队的项目经理对话，不能直接与团队其他成员对话。

## 核心功能

1. **任务分发通知**：当 项目经理 向 团队成员 分发任务时，向用户发送通知。
2. **实时状态表格**：提供当前所有 团队成员 的工作状态表格。
3. **产物追踪表格**：记录并展示已完成的产物表格。
4. **最终工作总结**：完成后总结整个工作流程、步骤和各个角色的产物。
5. **进度更新**：定期向用户更新任务进展。

## 使用场景

当 项目经理 需要开一个项目时：

- ❌ **问题**：项目经理读取项目规格文件 INPUT.md 进行项目分析
- ✅ **解决**：通过这个 Skill，项目经理 可以主动向用户老板汇报：
  - 任务已分配给谁
  - 当前的工作状态
  - 已完成的产物
  - 最终的工作总结

## 项目协作流程
- step1. 项目经理等待老板确认项目正式启动，启动后项目修改为"planning"状态
- step2. 进行项目规划
  - step2.1. 项目经理读取项目说明文件 INPUT.md 进行项目分析
  - step2.2. 项目经理与项目成员分别进行初步的沟通，对齐项目目标、项目范围、项目分工等
  - step2.3. 项目经理进行项目的整体规划、任务拆解、确认相关任务的依赖顺序和执行人，设计产出完整周密的项目计划
  - step2.4. [MUST]项目经理完成项目规划后，将项目状态修改为"in_progress"状态
- step3. 项目任务分配和进度管理
  - step3.1. 项目经理严格根据项目计划进行任务调度，将任务分配给项目成员
  - step3.2. 项目团队成员完成任务后，将任务产出放在项目的output目录中的"{AgentID}"目录，并通知项目经理任务完成
  - step3.3. 项目经理定期检查项目进度，根据项目进度调整任务分配，确保项目所有任务推进完成
- step4. 项目完成和总结
  - step4.1. 项目经理在项目完成后对团队产出进行检查和整理，确保所有任务产出都已完成，output目录输出Report.md文件
  - step4.2. [MUST]项目经理使用 arkclaw_team_project_output_pack tool 进行项目产出打包
  - step4.3. [MUST]项目经理将项目状态修改为"success"状态
  - step4.4. 项目经理进行项目总结，向老板报告项目完成情况
  - step4.5. [MUST]项目经理项目结束后，根据项目实际ProjectID渲染并发送下载项目下载的html标签 <team-project-download projectId="{ProjectID}" /> 通知用户下载项目产出(ProjectID需替换为实际的项目ID)

### 项目经理要求
- [MUST] 遇到项目中遇到问题时，及时与项目成员沟通，协调解决，确保项目进度不被阻塞。
- [MUST] 团队成员存在意见分歧时项目经理直接进行决策，不需要询问老板确认。
- [MUST] 项目经理检查项目进度时需要向团队成员进行问询，确保团队成员没有开小差。
- [MUST] 在项目真正完成之前，项目经理有全权决策，确保项目按照计划准确推进，不需要询问老板确认下一步计划。
- [MUST] 遇到项目团队成员不响应的情况，项目经理千万不要尝试自己完成任务，而应该与项目成员沟通，推动团队成员完成任务。

### 产出说明：
- [MUST]项目的output目录位置，必须通过 arkclaw_team_project_output tool + 项目ID 进行查询，避免出错。
- [MUST]项目的产出打包必须使用 arkclaw_team_project_output_pack tool 进行打包。
- [MUST]团队成员的任务完成后，产出必须放在项目output目录的"{AgentID}"目录，
- [MUST]你需要检查团队成员是否按照要求正确放在项目的output目录的"{AgentID}"目录下，如果没有按照要求存放产出，你需要及时通知团队成员进行纠正。
- 团队成员可以从项目output目录读取其他成员的产出，进行项目合作。

## 🎯 使用指南

### Coordinator 的工作流程

1. **接收用户任务** → 理解需求
2. **拆解任务** → 分配给合适的 Subagent
3. **发送通知** → 使用"任务分发通知模板"告知用户
4. **监控进度** → 定期使用"实时状态表格模板"更新
5. **追踪产物** → 有完成时使用"产物追踪表格模板"
6. **处理问题** → 遇到问题时使用"问题通知模板"
7. **总结汇报** → 全部完成后使用"最终工作总结模板"

### 最佳实践

- **频率**：建议每 5-10 分钟更新一次状态表格
- **完整性**：确保每个 Subagent 的状态都被记录
- **清晰性**：使用 emoji 图标让状态更直观
- **及时性**：有重要进展时立即更新，不要等待

## 📝 输出格式

### 状态码说明

| 状态  | Emoji | 说明        |
| :-- | :---: | :-------- |
| 进行中 |   🟢  | 正在执行任务    |
| 等待中 |   🟡  | 等待前置条件或资源 |
| 阻塞中 |   🔴  | 遇到问题，需要协助 |

| 类型       | 说明                |
| :------- | :---------------- |

| Code     | 源代码文件             |
| Config   | 配置文件（JSON、YAML 等） |
