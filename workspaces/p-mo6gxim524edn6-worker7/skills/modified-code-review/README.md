# Modified Code Review（修改代码评审）

对用户修改的代码进行评审，给出最佳实践建议、性价比分析与代码评分。

## What It Does

- 获取改动范围（git diff / PR / 指定文件）
- 多维度评审：正确性、可读性、可维护性、性能、安全、测试
- 最佳实践建议（必须改 / 建议改 / 可选优化）及示例
- 修改方案性价比（收益 vs 成本）
- 百分制代码评分与等级（A 优秀 ~ E 不达标）

## How to Use

当用户要求「review 我修改的代码」「审查修改」「代码评审」或需要「最佳实践」「性价比」「代码评分」时启用，输出《修改代码评审报告》。

## Requirements

- 基于用户提供的 diff/文件/上下文，不臆测未展示代码
